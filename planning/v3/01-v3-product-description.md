# Evolvo V3 Product Description

## Overview

Evolvo V3 is the next strategic evolution of Evolvo V2.

Evolvo V2 established the execution foundation:
- multi-project runtime
- project registry
- project lifecycle control
- orchestration and worker-based delivery
- infrastructure planning for self-hosted operation

Evolvo V3 upgrades that foundation into an autonomous product studio and portfolio operating system.

Instead of only executing approved project work, Evolvo V3 continuously:
- researches pain points across the web
- stores and structures evidence
- identifies concentrated problem areas
- generates SaaS ideas from validated pain
- scores ideas across multiple business and execution dimensions
- presents ideas for approval
- turns approved ideas into new projects
- builds those projects using the Evolvo V2 execution system
- deploys them to Hetzner
- prepares domain and pricing/subscription setup
- listens to post-launch feedback
- feeds validated feedback back into the delivery cycle
- reports on the entire system through an overarching Governor

## Product Goal

The goal of Evolvo V3 is to operate as Patrick’s private autonomous SaaS studio.

It should help Patrick:
- identify what is worth building
- avoid wasting time on weak opportunities
- launch products faster
- standardise deployment and commercialisation
- improve launched products continuously
- maintain visibility and control over the entire portfolio

## Primary User

The primary user is Patrick.

Initially, Evolvo V3 is an internal system used to:
- discover new product opportunities
- approve or reject those opportunities
- execute product delivery
- deploy and commercialise approved projects
- govern the health of the portfolio

In future, selected capabilities may also support client review and advisory workflows.

## Product Positioning

Evolvo V2 is the Execution OS.

Evolvo V3 is the Venture Studio OS.

V3 does not replace V2’s core execution engine.  
It builds on it and adds portfolio intelligence, research, scoring, aftercare, and governance.

## Core Capabilities

### 1. Portfolio Management
Tracks ideas, approved projects, live products, archived work, deployment state, commercial state, and feedback state.

### 2. Continuous Research
Collects and structures pain-point evidence from the web.

### 3. Opportunity Intelligence
Clusters pain, identifies patterns, and detects viable opportunity areas.

### 4. Idea Generation
Synthesises strong SaaS concepts from validated clusters of pain.

### 5. Multi-Dimensional Scoring
Scores ideas using explainable business, product, and execution criteria.

### 6. Approval-Gated Execution
Requires Patrick’s approval before critical actions such as:
- project creation
- domain purchase
- Stripe live object creation
- production deployment

### 7. Product Definition
Transforms approved ideas into execution-ready project definitions.

### 8. V2 Execution Reuse
Uses the Evolvo V2 engine to plan, build, test, refine, and deliver approved projects.

### 9. Deployment and Commercialisation
Deploys products to Hetzner and prepares domain and Stripe configuration.

### 10. Aftercare
Listens to feedback and routes validated signals back into the development cycle.

### 11. Governor
Watches the full system, critiques performance, identifies waste and risk, and reports portfolio-wide insight.

## Key Principles

### Evidence over intuition
Ideas must be grounded in traceable evidence.

### Approval-gated autonomy
The system can prepare and recommend, but critical real-world actions require approval.

### Launch is not the finish line
Post-launch improvement is part of the product lifecycle.

### Reuse the trusted execution core
V3 should reuse strong V2 capabilities wherever possible.

### The system must critique itself
The Governor must actively identify weak work, drift, waste, and missed opportunities.

## Non-Goals

The following are out of scope for V3 unless explicitly added later:
- making Evolvo itself a public multi-tenant SaaS
- allowing unapproved live financial or domain actions
- replacing the V2 execution engine without evidence that replacement is necessary
- fully autonomous production release without approval

## End State

When complete, Evolvo V3 should function as a private autonomous product studio that can:
- discover product opportunities
- recommend what to build
- execute approved work
- deploy and commercialise products
- learn from user feedback
- continuously govern and improve the whole portfolio
