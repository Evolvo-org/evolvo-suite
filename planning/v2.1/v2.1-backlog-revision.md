# Evolvo v2.1 Backlog Revision

## Purpose

This document defines the implementation and documentation changes required to apply the v2.1 planning model revision.

This is a follow-up revision document and should be used to update the existing v2 system and planning pack.

---

# Revision goal

Apply the v2.1 planning model across:

- planning docs
- backend domain model
- scheduler and orchestration logic
- web UI
- runtime execution flow
- agent definitions
- workflow handling

The goal is to remove idea generation, replace it with approved plan expansion, and make real AI agent execution explicitly runtime-owned.

---

# High-level change set

## Remove
- Inbox column
- inbox agent
- idea generation loop
- idea triage flow
- inbox scheduling logic
- idea acceptance/rejection UI where no longer relevant

## Add or revise
- planning expansion flow from supplied development plan
- plan approval flow
- approval-aware promotion to execution
- planning hierarchy as the first generated artifact
- plan-scope interpretation logic
- runtime-owned AI execution
- flat runtime model config for all active agent roles

# Acceptance criteria for the v2.1 revision

The revision is complete when:

- the system no longer depends on an inbox or idea-generation stage
- a supplied product description and development plan are the planning inputs
- the planning agent expands the full supplied scope into epics, tasks, and subtasks
- the generated backlog can be reviewed and approved
- only approved work may enter execution
- the dev agent begins only after approval
- planning, dev, review, and release execute through `apps/runtime`
- web, API, and scheduler all reflect the revised model
- outdated inbox concepts are removed or clearly deprecated
- there is one flat runtime file controlling agent model routing
