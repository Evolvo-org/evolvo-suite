# Evolvo v2.2 Backlog Revision

## Purpose

This document defines the implementation and documentation changes required to apply the v2.2 public website revision.

This is a follow-up revision document and should be used to update the existing v2, v2.1, and current system planning pack.

---

# Revision goal

Apply the v2.2 website layer across:

- web routing
- public content rendering
- authentication and route protection
- admin bootstrap access
- SEO and content delivery
- supporting docs

The goal is to add a public homepage and blog in the same Next.js app while keeping the real product behind protected routes.

---

# High-level change set

## Add
- public homepage
- public blog index
- public blog post routes
- protected `/app` route boundary
- login route handling
- admin bootstrap script
- markdown or MDX blog content pipeline
- sitemap/SEO support for public routes

## Revise
- route structure
- auth guard structure
- layout separation between public and protected areas
- navigation model
- documentation around access and entry points

## Do not add
- public signup flow
- signup messaging
- pricing conversion flows
- public product access
- CMS dependency for blog content

# Acceptance criteria for the v2.2 revision

The revision is complete when:

- the same Next.js app serves both public and protected experiences
- `/` is a public homepage
- `/blog` is a public blog index
- `/blog/[slug]` is a public blog detail route
- `/login` exists but is not publicly advertised
- protected product routes live under `/app`
- unauthenticated access to `/app` redirects to `/login`
- blog content is loaded from local markdown or MDX files
- a script exists to set or reset the admin email and password
- the public site positions Evolvo as an autonomous software factory
- no public signup language is present
